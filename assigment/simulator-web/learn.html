OSI & TCP/IP Layer Reference

A complete breakdown of how network communication is structured — from high-level applications to raw physical signals.

INTRO

Protocol Layering

📡 What is a Protocol? A protocol defines the rules all communicating parties must follow — like a language two machines agree to speak.
  • Simple communication needs only one protocol. Complex communication divides tasks into layers — one protocol per layer.
  • Protocol layering enables us to divide complex tasks into smaller, simpler tasks — each layer handles one concern.
  • Each layer acts as a black box — it takes inputs from the layer above, produces outputs for the layer below.
  • Each layer on the sending side uses the services of the layer directly below it.
  • Each layer on the receiving side passes data upward, stripping its own header/trailer at each step.
OSI MODEL

OSI Model Overview

🏛️ OSI = Open Systems Interconnection. Introduced in the late 1970s. Not a protocol — it is a model for designing flexible, robust, interoperable networks.
  • Has 7 layers, each defining part of the process of moving information across a network.
  • Peer-to-peer process: Processes on each machine communicate logically at the same layer level — even though data physically travels down one stack and up the other.
  • Each layer adds a header (encapsulation) while sending; strips it while receiving (decapsulation).

LAYER 7

Application Layer

TCP/IP: Application
🖥️ What it does: Enables users (human or software) to access the network. Provides user interfaces and support for services like email, file transfer, and the web.
  • This is the layer the user directly interacts with — web browsers, email clients, FTP programs.
  • Provides network services directly to end-user applications.
  • Responsible for identifying the communication partner and checking resource availability.
HTTP HTTPS FTP SMTP DNS TELNET SSH SNMP
LAYER 6

Presentation Layer

TCP/IP: Application
🔤 What it does: Concerned with the syntax and semantics of information exchanged between two systems. Converts data into a format the application layer can accept.
  • Translation: Converts between different character encoding systems (e.g., ASCII ↔ EBCDIC).
  • Compression: Reduces the number of bits to be transmitted, improving bandwidth efficiency.
  • Encryption: Transforms data to ensure privacy during transmission; decryption on the other end.

LAYER 5

Session Layer

TCP/IP: Application
🤝 What it does: The network dialog controller. Establishes, maintains, and synchronizes the interaction between communicating systems.
  • A network session is a temporary interactive information interchange between two or more communicating devices.
  • Authentication & Authorization — verifies who is talking and what they're allowed to do.
  • Dialog control — determines which side transmits at any given time (half-duplex or full-duplex).
  • Synchronization — inserts checkpoints into data streams so that if a failure occurs, only the data after the last checkpoint needs to be retransmitted.

LAYER 4

Transport Layer

TCP/IP: Transport
🔀 What it does: Responsible for process-to-process delivery of the entire message. The Network layer handles individual packets — the Transport layer handles the complete message.
  • Service-point addressing: Uses port numbers to direct data to the correct process on a machine.
  • Segmentation and reassembly: Divides the message into segments (each with a sequence number) and reassembles them at the destination.
  • Connection control: Can be connection-oriented (TCP — reliable, ordered) or connectionless (UDP — fast, unreliable).
  • End-to-end flow control: Prevents the sender from overwhelming the receiver.
  • Error control: Ensures the entire message arrives, requesting retransmission of corrupted or lost segments.
TCP — Connection-oriented UDP — Connectionless SCTP — Multimedia streams
LAYER 3

Network Layer

TCP/IP: Internet (Network)
🌐 What it does: Responsible for source-to-destination delivery of a packet across multiple networks. The Data Link layer handles frames on a single link; the Network layer routes across many links.
  • Logical addressing: Adds source and destination IP addresses to each packet so it can traverse multiple networks.
  • Routing: Determines the best path for packets to travel from source to destination through intermediate routers.
  • IP (Internet Protocol): Connectionless — defines the datagram format and how packets are routed. Best-effort delivery (no guarantee).
IP — Internet Protocol ICMP — Error reporting ARP — Address resolution DHCP — Dynamic addressing IGMP — Multicasting
LAYER 2

Data Link Layer

TCP/IP: Network Access
🔗 What it does: Responsible for moving frames from one hop (node) to the next. Transforms the raw physical layer into a reliable link. Makes the physical layer appear error-free to upper layers.
  • Framing: Divides the bitstream into manageable data units (frames) with a defined start/end boundary.
  • Physical addressing: Adds the sender's and receiver's MAC address to each frame for local delivery.
  • Hop-to-hop delivery: Gets a frame from one node (computer or router) to the adjacent node on the same network.
  • Flow control: Prevents a fast sender from flooding a slow receiver on the same link.
  • Error control: Detects and retransmits damaged or lost frames (e.g., using CRC checksums).
  • Access control: Determines which device has control of the shared transmission medium at any given time.
LAYER 1

Physical Layer

TCP/IP: Network Access
⚡ What it does: Responsible for movement of individual bits from one hop to the next. Deals with the mechanical and electrical specifications of the transmission medium.
  • Data rate: Defines how many bits per second can be transmitted.
  • Bit synchronization: Ensures sender and receiver agree on the bit duration (clock synchronization).
  • Line configuration: Point-to-point or multipoint connections.
  • Physical topology: The layout of the network — mesh, star, ring, bus.
  • Transmission mode: Simplex, half-duplex, or full-duplex communication.

TCP/IP

TCP/IP Protocol Suite

🌍 TCP/IP = Transmission Control Protocol / Internet Protocol. The protocol suite actually used on the Internet today. Originally defined as 4 layers — today conceptually described as 5 layers.
  • A protocol suite is a set of protocols that work together, each handling a specific layer of communication.
  • TCP/IP was designed to be robust — the network can survive partial damage and still route packets.
  • Unlike OSI, TCP/IP is a real implementation — OSI is a theoretical model.

TCP/IP LAYERS

TCP/IP — Layer by Layer

📋 Each TCP/IP layer has specific protocols and responsibilities. Unlike OSI's rigid separation, TCP/IP layers are more flexible and overlap in practice.
5
APPLICATION LAYER
Process-to-process communication. Defines how applications access network services.
HTTP — Web SMTP — Email FTP — File Transfer DNS — Domain Names SSH TELNET SNMP
4
TRANSPORT LAYER
TCP — Connection-oriented. Provides flow control, error control, and congestion control. Guaranteed delivery.
UDP — Connectionless. Small overhead, no guarantee. Best for real-time apps (video, gaming).
SCTP — Designed for multimedia applications (e.g., VoIP).
3

Moves data across a single link. Encapsulates datagrams into frames. Provides error detection and correction on individual hops.
1

Carries individual bits across the physical link. Transforms bits into signals appropriate for the transmission medium — electrical, optical, or radio.
ADDRESSING

Addressing in TCP/IP

📍 TCP/IP uses three types of addresses — one per layer level — to uniquely identify machines, processes, and physical hardware at different scopes.
🔌
PHYSICAL ADDRESS (MAC)
Also called the link address. Defined by the LAN or WAN technology in use. Identifies a specific network interface card (NIC) on the local network. Changes as a frame hops from router to router.
Example: 48-bit Ethernet MAC → AA:BB:CC:DD:EE:FF
🌐
LOGICAL ADDRESS (IP)
Necessary for universal communication that must cross multiple networks. A 32-bit (IPv4) address uniquely identifies a host globally. Also called the IP address.
Example: 32-bit IPv4 → 192.168.1.100
🔢
PORT ADDRESS
Used for process-to-process communication. A 16-bit number in TCP/IP that identifies which application or service should receive the data. Allows multiple processes to share a single IP at the same time.
TELNET: port 23 | FTP: port 21 | HTTP: port 80
